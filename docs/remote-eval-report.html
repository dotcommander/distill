Date: 2026-06-24 · Decision: public benchmark leaderboards are now the PRIMARY signal for choosing models per role; distill's local evals are the SECONDARY / fallback signal.

This report records the cross-reference that drove the new per-role config defaults, the public boards to follow, and the code that wires it.

TL;DR

  • Selection is now remote-primary: pick the best-ranked model among the cost-capped OpenRouter roster (runs/roster.sh, ≤ $8 / 1M out) on each role's closest public leaderboard.
  • The public boards independently corroborate distill's own merit tournament: z-ai/glm-5.2 is the #1 prose model in the roster on EQ-Bench, matching the local result.
  • Faithfulness boards reveal a split a single model can't exploit — research/fuse want the least-hallucinating small model (gemini-2.5-flash-lite, HHEM 3.3%), while write/edit want the best prose model (glm-5.2). The digest pipeline now uses per-role models.
  • Local evals remain the fallback for roles/models the public boards don't cover (the FALLBACK rule).

FALLBACK rule (when local evals apply)

  • Roster model absent from the board → score it locally (distill eval facts, distill tournament/panel).
  • Board inconclusive (within noise) → tiebreak with the aligned local harness.
  • Role has no public board (label/sentiment) → route on cost + latency; local macro-F1 as a floor.
